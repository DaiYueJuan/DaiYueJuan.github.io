HEXO的Next主题设置以及个性化设置

发表于 2025-07-05 更新于 2025-07-06 分类于 hexo 阅读次数： 本文字数： 3k 阅读时长 ≈ 3 分钟
我的hexo界面设置

在已经搭建Hexo个人博客的基础上，设置next的主题和个性化设置。

搭建Hexo个人博客可以参考Fiveth的博客“如何用Hexo搭建个人博客”。如何用Hexo搭建个人博客?

1.设置next主题

进入hexo官网，点击“主题”，在搜索的地方输入“next”，点击“nexT”进入到github里面。下面有官方的详细安装介绍。

我使用的是 clone ssh的方法。将文件夹放在themes目录下。

然后在 _config.yml 文件设置theme: next

最后hexo clean && hexo g && hexo s

即可生成

注意：这里的theme: next应该与文件夹的名称相同，如果下载的文件是hexo-theme-next，可以改文件夹的名称也可以改theme: next为 theme: hexo-theme-next

2.个性化next主题

根据clone下来的next文件里面的 _config.yml 可以得知

===============================================================

It’s recommended to use Alternate Theme Config to configure NexT

Modifying this file may result in merge conflict

See: https://theme-next.js.org/docs/getting-started/configuration

===============================================================

Theme Core Configuration Settings

See: https://theme-next.js.org/docs/theme-settings/

进入网站可以知道，NexT 目前支持两种推荐的配置方式，你只需要选择其中一种：

​ 传统方式是修改 themes/next/_config.yml，但这个文件会在更新主题时被覆盖，所以 NexT 主题现在推荐使用 Hexo 的“主题配置重载（Alternate Theme Config）”功能。

​ 它允许你在 Hexo 根目录创建一个 _config.[theme_name].yml 文件（NexT 默认的 theme 名是 next，所以就是 _config.next.yml）。

​ Hexo 会优先加载这个文件的配置，并覆盖掉默认的主题配置。

​ theme_config 模式 —— 把 NexT 配置直接写在主配置文件 _config.yml 中。

我使用的第一种方法。将其copy即可。
