    解决 Typora 插入图片路径与 Hexo 不兼容

    发表于 2025-09-11 分类于 hexo 本文字数： 702 阅读时长 ≈ 1 分钟
    在typora里面编辑图片之后自动生成的路径与hexo识别的路径不兼容，每次都要手动修改很麻烦，于是找到解决方法，一劳永逸。

    问题描述：

    每次在typaro 里面插入的图片格式都是这样的绝对路径

    而在hexo显示需要的是这样的格式

    第一个是绝对路径应该改为相对路径

    第二个是 / 与 \ 的不同

    解决方法：

    核心思路：让 Typora 自动将图片保存到 Hexo 文章的同名资源文件夹，并生成相对路径。

    1. 配置 Hexo 资源文件夹

    此后，使用 hexo new "文章标题" 创建文章时，会自动生成同名文件夹（如 相机标定原理介绍/）。

    2. 配置 Typora 自动保存图片

    打开 Typora → 偏好设置 → 图像，按以下步骤操作：

    • 插入图片时：选择 复制到指定路径，并输入路径：

      • 若文章名为 相机标定原理介绍.md，图片会自动保存到 相机标定原理介绍/ 文件夹。
      • 关键：路径必须以 ./ 开头，表示当前文章所在目录。

    • 勾选：对本地位置的图片应用上述规则。

    • 链接格式：选择 相对路径（确保生成的路径不带盘符或绝对路径）。

    3. 验证效果

    • 插入图片时，Typora 会自动将图片复制到文章同名文件夹，并生成路径：

    • 执行 hexo g && hexo d 部署后，图片会被正确打包到 Hexo 的 public 目录。

    4. 注意事项

    • 必须通过 hexo new 命令创建文章，否则不会自动生成资源文件夹。
    • 若手动创建文章，需手动在 source/_posts/ 下创建同名文件夹，并在 Typora 中粘贴图片时选择 复制到当前文件夹。
