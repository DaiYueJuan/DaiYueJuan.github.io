Crash Course Computer Science学习笔记

发表于 2025-10-09 更新于 2025-10-10 分类于 Study Notes 阅读次数： 本文字数： 17k 阅读时长 ≈ 15 分钟
Crash Course Computer Science是计算机科普类型的系列视频，科普计算机的历史以及计算机运作的原理等等，作为我非科班生基础的补充。

前言

在我的研究生阶段，我开始学习编程，通过编程去参与实验室项目，对软件进行增删查改也使用设计模式去设计模块融入软件。但是我从未系统的了解过计算机的历史。 Crash Course Computer Science非常适合现在的我，它简单有趣、通俗易懂。感谢课程制造者以及字幕组。互联网的世界真是丰富多彩、妙不可言。

在看完Crash Course Computer Science 的课程之后，我有一种豁然开朗的感觉。了解到计算机“成长”的历史，根据需求的不断创新以及在数学、科技、物理发展的促进下有了今天互联网的时代。

从为了解决美国人口统计问题到如今离不开计算机，从使用齿轮–继电器控制电路–二极管三极管–晶体三极管，我们站在前人的肩膀上感受和体验科技发展带来的便利，同时也使我思考What can I do？事物的发展都不是一蹴而就的，是由许多微小的变化积累而来的，我也可以去解决一些小的问题，发展就是为了解决当下遇见的问题的。用计算机去处理重复性的劳动。

抽象思想的学习和运用。 a new level of abstraction :)

在这个课程之后我也拨开了计算机科学神秘的面纱，对其有了系统性的认识，虽然只是一个大概的认识。而我将继续利用互联网学习我感兴趣以及我需要系统性学习的知识。

Crash Course Computer Science简介

在短短 8 个小时里非常生动且全面地科普了关于计算机科学的方方面面：计算机的历史、计算机是如何运作的、组成计算机的各个重要模块、计算机科学中的重要思想等等等等。Crash Course Computer Science（计算机科学速成课）共 40 集，整体遵循 “从底层原理到上层应用，从技术本身到社会影响” 的逻辑脉络。整个教程遵循 “从‘是什么’（基础）→‘怎么做’（硬件 / 软件 / 算法）→‘连起来’（网络）→‘用在哪 / 怎么办’（应用 / 社会） ” 的认知路径，从最底层的二进制到最前沿的量子计算，既覆盖计算机科学的核心知识点，也兼顾历史背景与现实意义，适合零基础学习者建立完整的计算机科学框架。

总结

第一部分：计算机基础与历史

本模块旨在回答 “什么是计算机” ，“计算机如何诞生”，搭建最基础的认知框架，核心是 “二进制” 与 “早期计算逻辑”。

  1. 计算机的本质与历史溯源
    • 计算机的定义：能执行 “输入→处理→输出→存储” 循环的设备（不止电脑，手机、微波炉均是）
    • 早期计算工具：算盘（手动计算辅助）、差分机（巴贝奇，首台机械计算雏形）、穿孔卡片机（用于人口普查等批量数据处理）
    • 图灵与图灵机：图灵的核心贡献 ——“通用计算” 思想（任何可计算问题都能通过一套规则在抽象机器上实现）
    • ENIAC：世界首台通用电子计算机（1946 年，用于二战弹道计算，体积庞大、功耗高）
  2. 二进制与信息表示
    • 二进制的必要性：计算机硬件（晶体管）仅能识别 “通 / 断”（0/1），二进制是硬件与信息的桥梁
    • 二进制与十进制、十六进制的转换逻辑（重点：位权概念）
    • 信息的二进制编码：文本（ASCII、Unicode）、图像（像素与二进制颜色值）、声音（采样率与二进制振幅）

第二部分：计算机硬件原理（物理基础）

本模块拆解计算机的 “身体”，从最底层的 “晶体管” 到 “整机组件协同”，解释 “硬件如何实现计算”。

  1. 基础电子组件：从晶体管到逻辑门
    • 晶体管：计算机的 “原子”—— 半导体材料制成，可实现 “开关” 功能（控制电流通断，对应 0/1）
    • 逻辑门：由晶体管组成的 “计算单元”—— 与门（AND）、或门（OR）、非门（NOT）、异或门（XOR），实现基础逻辑运算
    • 组合逻辑电路：多个逻辑门组合，实现更复杂运算（如加法器：半加器→全加器→ ripple carry adder）
  2. 核心硬件组件与功能
    • CPU（中央处理器）：计算机的 “大脑”
      • 结构：ALU（算术逻辑单元，执行加减乘除 / 逻辑运算）、控制单元（协调指令执行顺序）、寄存器（临时存储数据 / 指令，速度极快）
      • 指令周期：取指（从内存拿指令）→译码（理解指令）→执行（ALU 运算）→写回（结果存回寄存器 / 内存）
    • 内存与存储：计算机的 “记忆”
      • RAM（随机存取存储器）：临时存储，断电数据丢失，速度快（配合 CPU 高速读写）
      • ROM（只读存储器）：永久存储，断电数据不丢（如 BIOS，启动计算机的基础指令）
      • 外部存储：硬盘（机械，容量大、速度慢）、SSD（固态硬盘，基于闪存，速度快、成本高）
    • 输入 / 输出（I/O）设备：计算机与外界的 “接口”
      • 输入：键盘、鼠标、摄像头（将物理信号转为电信号 / 二进制）
      • 输出：显示器、打印机、音箱（将二进制转为物理信号）
    • 总线（Bus）：硬件组件的 “高速公路”—— 地址总线（指定数据位置）、数据总线（传输数据）、控制总线（传递控制信号，如 “读 / 写” 命令）
  3. 硬件发展规律：摩尔定律
    • 核心内容：每 18-24 个月，集成电路上可容纳的晶体管数量翻倍，性能也翻倍
    • 影响：推动计算机从 “房间大小” 缩小到 “手掌大小”（手机），成本持续降低
    • 挑战：物理极限（晶体管尺寸接近原子级，漏电问题凸显）

第三部分：软件系统（控制硬件的 “灵魂”）

本模块聚焦 “软件如何指挥硬件工作”，从操作系统到编程语言，解释 “抽象层如何降低使用门槛”。

  1. 操作系统（OS）：硬件与用户的 “中间层”
    • 核心功能：进程管理（多任务调度，如同时打开浏览器 + 音乐）、内存管理（分配 RAM 空间，避免冲突）、设备管理（驱动程序，让 OS 识别打印机 / 显卡）、文件管理（组织硬盘数据，如文件夹、文件格式）
    • 主流 OS：Windows（桌面端）、macOS（桌面端）、Linux（开源，服务器 / 嵌入式常用）、Android/iOS（移动端）
    • 启动流程：BIOS→引导程序→加载 OS 内核→启动用户界面
  2. 编程语言与编译原理
    • 编程语言的演进：从 “机器语言” 到 “高级语言”
      • 机器语言：二进制指令（直接控制硬件，难写难读）
      • 汇编语言：用英文缩写代替二进制（如 ADD 代表加法，仍依赖硬件架构）
      • 高级语言：接近人类自然语言（如 Python、Java、C++），不依赖硬件，跨平台性强
    • 编译与解释：将高级语言转为机器语言的工具
      • 编译器：一次性将整个高级语言程序编译为机器码（如 C++，运行速度快）
      • 解释器：逐行解释高级语言代码并执行（如 Python，开发效率高，运行速度慢）
    • 编程范式：面向过程（按步骤执行，如 C）、面向对象（以 “对象” 为核心，如 Java，封装 / 继承 / 多态）
  3. 软件的层级结构：从内核到应用
    • 底层：内核（OS 的核心，直接操作硬件）
    • 中层：系统调用（应用程序调用 OS 功能的接口，如 “打开文件” 需调用 OS 的文件管理函数）
    • 上层：应用程序（如浏览器、Office，用户直接操作的软件）

第四部分：数据结构与算法（计算机解决问题的 “方法”）

本模块是计算机科学的 “核心思维”，回答 “如何高效存储数据” “如何快速解决问题”，核心是 “效率”。

  1. 数据结构：组织数据的 “容器”
    • 线性结构：数据按顺序排列
      • 数组（Array）：连续内存空间，随机访问快（按索引找数据），插入 / 删除慢（需移动元素）
      • 链表（Linked List）：非连续内存，通过指针连接，插入 / 删除快，随机访问慢
      • 栈（Stack）：“先进后出”（如浏览器后退、函数调用栈）
      • 队列（Queue）：“先进先出”（如打印机排队、消息队列）
    • 非线性结构：数据多对多关联
      • 树（Tree）：分层结构，树是单向的，由根节点指向子节点。如二叉树（每个节点最多 2 个子节点）、二叉搜索树（左子树 < 根 < 右子树，查找快）、堆（用于排序、优先队列）
      • 图（Graph）：由节点和边组成（如社交网络、地图路径），分有向图 / 无向图、加权图 / 无权图。图是可以有双向指向的。
  2. 算法基础：解决问题的 “步骤”
    • 基础算法：
      • 排序算法：冒泡排序（简单但低效）、选择排序、插入排序、快速排序（分治思想，高效）、归并排序（稳定，适合大数据）
      • 搜索算法：线性搜索（遍历所有元素，适用于无序数据）、二分搜索（仅适用于有序数据，效率高）
    • 算法复杂度：衡量 “效率” 的标准
      • 时间复杂度：算法执行时间随数据量增长的趋势（如 O (1) 常数、O (log n) 对数、O (n) 线性、O (n²) 平方）
      • 空间复杂度：算法占用内存随数据量增长的趋势
    • 常用算法思想：分治（大问题拆小问题）、递归（函数调用自身，如斐波那契数列）、动态规划（存储子问题结果，避免重复计算，如背包问题）

第五部分：网络与互联网（计算机的 “连接”）

本模块解释 “计算机如何互联互通”，从局域网到互联网，拆解网络协议与架构。

  1. 网络基础：从局域网到广域网
    • 网络的定义：多台计算机通过通信设备（网卡、路由器）连接，实现数据共享
    • 局域网（LAN）：小范围网络（如家庭 WiFi、公司内网），常用以太网（Ethernet）协议
    • 广域网（WAN）：大范围网络（如跨城市连接），依赖光纤、卫星等长途传输介质
  2. 互联网（Internet）的核心：协议与架构
    • 互联网的诞生：ARPANET（1969 年，美国军方项目，为抗故障设计的分布式网络）
    • TCP/IP 协议簇：互联网的 “交通规则”
      • IP（网际协议）：负责 “寻址”—— 给每台设备分配 IP 地址（如 IPv4：192.168.1.1），确定数据传输的 “目的地”
      • TCP（传输控制协议）：负责 “可靠传输”—— 将数据拆分为数据包，确认接收（丢包重传），按顺序重组
      • HTTP/HTTPS：应用层协议，用于浏览器与服务器通信（HTTPS 比 HTTP 多了加密，更安全）
    • DNS（域名系统）：“IP 地址的翻译官”—— 将易记的域名（如www.baidu.com）转为 IP 地址（如 180.101.49.12），避免用户记忆复杂 IP
  3. 网络安全基础
    • 威胁类型：黑客攻击（如 DDOS：洪水般请求瘫痪服务器）、病毒（自我复制破坏数据）、钓鱼（伪装合法网站窃取信息）
    • 防御手段：加密（如 SSL/TLS，保护数据传输）、防火墙（过滤非法网络请求）、密码哈希（存储密码时不存明文，存哈希值）

第六部分：应用领域与社会影响（技术的 “落地与责任”）

本模块将前面的技术知识与实际应用结合，并探讨技术带来的社会问题与未来趋势。

  1. 计算机的核心应用领域
    • 数据库：存储海量结构化数据的系统
      • 关系型数据库（如 MySQL、SQL Server）：用表格（行 / 列）存储，支持 SQL 查询，适合交易类数据（如银行转账）
      • 非关系型数据库（如 MongoDB、Redis）：灵活存储非结构化数据（如社交软件的用户动态），适合高并发场景
    • 人工智能（AI）基础：
      • 机器学习：让计算机从数据中 “学习” 规律（如垃圾邮件识别、推荐系统）
      • 神经网络：模拟人脑神经元结构，实现深度学习（如图像识别、语音助手）
    • 计算机图形学：生成 / 处理图像的技术（如 3D 游戏渲染、电影特效，核心是 “像素计算” 与 “三维建模”）
    • 嵌入式系统：嵌入到设备中的专用计算机（如智能手表、汽车控制系统，体积小、功能专一）
  2. 技术的社会影响与伦理
    • 隐私问题：数据收集（如 APP 获取位置 / 通讯录）、数据滥用（如精准广告、身份泄露）
    • 数字鸿沟：不同地区 / 人群在获取计算机 / 网络资源上的差距（如农村与城市的网络覆盖率）
    • AI 伦理：算法偏见（如招聘 AI 因训练数据偏向男性而歧视女性）、自动化替代工作（如工厂机器人替代工人）
  3. 未来趋势：突破现有极限
    • 量子计算：基于量子力学（叠加态、纠缠态），解决传统计算机难以处理的问题（如密码破解、药物研发）
    • 量子互联网：利用量子纠缠实现 “绝对安全” 的通信（任何窃听都会改变量子状态，可被察觉）
    • 边缘计算：将计算任务从 “云端” 转移到 “设备端”（如智能家居、自动驾驶），降低延迟，保护隐私
